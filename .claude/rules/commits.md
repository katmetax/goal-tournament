## Commits

Format: `type(scope): subject`
Types: `feat` `fix` `refactor` `test` `docs` `chore` `style` `perf`
- Subject: ≤72 chars, imperative mood (`add X`, not `added X`), no trailing period
- Body (optional): explain *why*, not *what*; wrap at 72 chars
- Breaking changes: add `BREAKING CHANGE: description` footer

## Pull Requests

**Title**: same format as commit subject

**Body**:

## Summary
- What changed and why (2–3 bullets)

## Test plan
- [ ] Steps to verify the change
- [ ] Edge cases checked

## Notes
Breaking changes, migrations, deploy steps — omit section if none
